## Objective

Prove the shared architecture supports a second, compressed format without
special-casing it. SPZ import is the deliverable; the architectural claim is
that a new format reaches USD through the same `GaussianCloudData` →
`GaussianLayerWriter` path PLY uses, contributing no format-specific USD
authoring.

## What shipped

*Fill from the changelog at tag time. Delivered so far:*

- **SPZ container reader** (`plugins/gaussian-spz/src/io/SpzReader.*`).
  Versions 1-3, gzip-framed, implemented from Niantic's MIT-licensed
  specification rather than by vendoring the reference implementation.
  miniz 3.0.2 vendored at `third_party/miniz` because the runtime ships no
  zlib; gzip framing is parsed in-repo. `CanRead()` is a bounded inflate of
  the magic only.
- **Semantic decoder** (`GaussianSpzDecoder`). Dequantization per
  [SPZ_MAPPING.md](../reference/SPZ_MAPPING.md), the RUB→RDF reference-frame
  conversion (position and quaternion-vector negation plus per-coefficient SH
  band sign flips), both rotation encodings, and SH degrees 0-3.
- **USD integration.** The SPZ plugin routes decoded data through the shared
  `libs/gaussian-usd` writer; PLY and SPZ author an identical hierarchy,
  schema, and metadata.
- **Stable `GSPZ-****` diagnostics** with a machine-readable catalog,
  cross-checked against the source constants in both directions. Malformed,
  unsupported, and internal failures stay distinct: v4 and SH degree 4 are
  *unsupported*, not corrupt.
- **Real-asset corpus.** Two CC0-1.0 author-captured Scaniverse exports
  (`yashica-t4`, `leica-sofort`), 8,192 Gaussians each, derived by
  `scripts/spz_subset.py` with recorded provenance.
- **Performance baselines** for SPZ recorded in
  [PERFORMANCE_BASELINES.md](../reference/PERFORMANCE_BASELINES.md).
- **CI coverage.** Three `gaussian-spz` source cells mirroring the PLY ones,
  moving the PR and release lanes together.

## Compatibility

- SPZ container versions 1, 2, and 3. Version 4 (ZSTD, per-attribute streams)
  is rejected with a specific unsupported-version diagnostic, not a
  malformed-container one, and is deferred to v0.5.0.
- SH degrees 0-3. Degree 4 is rejected as unsupported (the shared model carries
  0-3).
- Read-only. No SPZ writing or export.
- PLY behaviour from v0.2.0 is unchanged; `GSPLY-****` codes are neither
  renumbered nor reused.

## Known limitations at this release

- **No PLY/SPZ equivalence fixtures.** The RUB→RDF conversion and SH sign
  flips are pinned by hand-encoded fixtures and an encode→decode inverse
  check, not by cross-format comparison against an independently produced
  asset. This is the largest remaining verification gap and is the reason the
  corpus assets must not be diffed against their PLY namesakes — they are
  independent captures, not two encodings of one reconstruction.
- **Windows stays capped at OST L4**, inherited from the hosted multiline-USDA
  line-ending finding, so the golden roundtrip gate runs only on macOS and
  Linux.
- **Package-origin L5 still skips** — upstream OST packaging seam.
- **macOS across-run package digest reproducibility is still open**; Windows
  and Linux archives are byte-identical across hosted runs.
- **The model's reference frame is PLY-native RDF while the authored stage
  declares `upAxis = "Y"`.** Correcting that is a major-version change and is
  tracked in the backlog, deliberately not fixed here.
- **Rendering is not included.** These plugins import; a Gaussian-capable
  Hydra delegate is required to see splats.
