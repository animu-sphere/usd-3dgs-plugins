# v0.4.0 — Gaussian Import Foundation

- **Tag:** `v0.4.0`
- **Commit:** `312764a1ea716505e98d3f969a816569eea41552`
- **Date:** 2026-07-22
- **Status:** published 2026-07-22 after human review

## Objective

Turn the decoder-to-USD seam proven by PLY and SPZ into a formal, documented,
reusable contract before a third format depends on it (design policy §7.4).
This release adds no end-user file format; it formalizes the existing
`format reader → semantic decoder → GaussianCloudData → GaussianLayerWriter →
ParticleField3DGaussianSplat` pipeline so SOG (v0.5.0) and later decoders can
be added without format-specific USD authoring, duplicated validation,
inconsistent coordinate handling, or incompatible diagnostics.

## What shipped

Matches the finalized `## [0.4.0]` section of
[CHANGELOG.md](../../CHANGELOG.md); no further entries are expected.

- **Normative model contract.**
  [GAUSSIAN_MODEL_CONTRACT.md](../reference/GAUSSIAN_MODEL_CONTRACT.md) is
  rewritten as the contract a decoder targets without reading PLY or SPZ code:
  decoded physical values only, strictly positive linear scales, sigmoid
  opacity, normalized scalar-first quaternions, Gaussian-major SH with fixed
  length relationships, supported degrees 0-3, the maximum-count and overflow
  policy, and the retained-source-metadata rules.
- **Authored frame corrected to RUB (Y-up)**
  ([ADR 0001](../adr/0001-model-frame-is-rub.md)). The canonical model frame
  moves from PLY-native RDF to RUB, matching the `upAxis = "Y"` every stage
  already declared. The PLY decoder converts RDF→RUB through the shared
  `FlipYZAxes`; SPZ is natively RUB and converts nothing. This is a pre-1.0
  correction to v0.1.0-v0.3.0 authored output: positions, orientation vector
  parts, and non-DC SH coefficients negate Y/Z, and extents move accordingly.
- **Shared semantic validation and size math** in `gaussianCore`. The runtime
  gate enforces the SH degree ceiling (`kMaxShDegree`) and quaternion
  normalization; `GaussianSizeMath.h` provides overflow-checked size
  arithmetic and `TryResize`, so both decoders allocate through one
  implementation (failure diagnostics `GSPLY-E018` / `GSPZ-E014`). The
  authored-extent computation moved into `ComputeCloudExtent` so the writer
  and the test kit share one implementation; authored extents are unchanged.
- **Decoder test kit** (`openstrata/gs/testing/DecoderTestKit.h`): canonical
  expected models, tolerance-aware comparison with quaternion sign
  equivalence and a derived-extent check, and model-level invalid cases — a
  decoder tested against the contract without authoring a USD stage. A mock
  decoder (`test_decoder_kit.cpp`) demonstrates the full loop using no PLY or
  SPZ code, the release's "a third decoder needs only the contract" criterion
  held by a test.
- **Import-statistics seam** (`GaussianImportStats`): one shared per-import
  record (source format/version, counts, byte sizes, model-frame bounds, and
  reader/decode/authoring times), filled by both decoders and emitted as one
  stable line under `TF_DEBUG=GSPLY_IMPORT_STATS` / `GSPZ_IMPORT_STATS`.
  Collection is skipped when the flags are off; the authored stage is
  unchanged.
- **Header / API boundary documented**
  ([API_BOUNDARY.md](../architecture/API_BOUNDARY.md)): every installed header
  classified public / contributor / internal / plugin-private, with the
  end-user compatibility surface fixed as the authored stage, the file-format
  arguments, and the diagnostic codes — not a C++ header, and nothing
  ABI/API-frozen before v1.0.0.
- **Contributor guide**
  ([ADDING_A_FORMAT_DECODER.md](../contributing/ADDING_A_FORMAT_DECODER.md)):
  the end-to-end path for a new format, generalizing the PLY-specific loop to
  the shared contract, followable with no PLY or SPZ source.
- **`gaussian-sog` skeleton.** A third bundle that registers `.sog` and
  rejects it with the stable `GSSOG-E001` not-implemented diagnostic (never an
  empty stage). It builds, tests, and packages through the same declarative
  path as the shipping bundles and gained three PR CI cells by editing
  `openstrata.ci.yaml` alone; its cells carry the new `publish: never` marker,
  so **the skeleton ships in no release package**. The v0.5.0 SOG plan and
  dependency decisions are recorded in
  [SOG_FORMAT.md](../reference/SOG_FORMAT.md).

## Compatibility

- **Authored orientation changed (breaking, pre-1.0).** Stages imported by
  v0.4.0 are oriented RUB / Y-up to match their declared `upAxis`; stages
  imported by v0.1.0-v0.3.0 were RDF and rendered flipped in `upAxis`-honoring
  consumers. Re-import to update; the migration is described in
  [ADR 0001](../adr/0001-model-frame-is-rub.md).
- **No new end-user format.** `.ply` (Graphdeco, versions per
  [PLY_DIALECTS.md](../reference/PLY_DIALECTS.md)) and `.spz` (versions 1-3)
  read exactly as in v0.3.0, now through the formalized contract. `.sog` is
  recognized but rejected (`GSSOG-E001`); SOG import is v0.5.0.
- **Diagnostics.** New codes `GSPLY-E018`, `GSPZ-E014` (model
  overflow/allocation), and the `GSSOG-****` namespace. Existing
  `GSPLY-****` / `GSPZ-****` codes are neither renumbered nor reused.
- Read-only. No writing or export.

## Known limitations at this release

- **Authored orientation changed relative to v0.1.0-v0.3.0** (RUB / Y-up,
  above). A local viewer script still pointing at a packaged v0.1.0-v0.3.0
  build will render the old orientation until repointed at a v0.4.0 build.
- **`.sog` is recognized but not decoded.** The skeleton rejects every `.sog`
  with `GSSOG-E001`; SOG v2 import is the v0.5.0 theme.
- **SPZ version 4 (ZSTD)** is still rejected with a specific
  unsupported-version diagnostic; reconsidered after SOG v2 ships.
- **Windows stays capped at OST L4**, inherited from the hosted multiline-USDA
  line-ending finding, so the golden roundtrip gate runs only on macOS and
  Linux.
- **Package-origin L5 still skips** — upstream OST packaging seam.
- **macOS across-run package digest reproducibility is still open**; Windows
  and Linux archives are byte-identical across hosted runs.
- **Rendering is not included.** These plugins import; a Gaussian-capable Hydra
  delegate ([`hydra-merlin`](https://github.com/animu-sphere/hydra-merlin)) is
  required to see splats.
