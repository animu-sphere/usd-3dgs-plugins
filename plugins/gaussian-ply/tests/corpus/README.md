# Real-asset corpus

Small subsets of real trained Gaussian Splatting assets, committed for
tolerance-based semantic tests. Synthetic fixtures in
[`../fixtures`](../fixtures) remain the primary, exact-assertion coverage;
corpus assets supplement them and must never be the sole source of coverage
(design policy §17).

## Admission rules

An asset enters this directory only when all of the following hold:

1. **License.** The asset is redistributable and modifiable without
   field-of-use restrictions: CC0, CC-BY (with attribution recorded in the
   asset's `PROVENANCE.md`), or an original work dedicated by its author.
   Non-commercial, no-derivatives, and unclear licenses are excluded; large
   references with such terms stay outside the repository as
   download-on-demand data.
2. **Provenance.** The asset directory records the full chain — capture,
   trainer (name, version, license, seed, step count), source export hash —
   in a human-readable `PROVENANCE.md`, and the derivation is captured
   machine-readably in `<asset>.ply.provenance.json`.
3. **Deterministic derivation.** The committed file is produced from its
   recorded source by [`scripts/ply_subset.py`](../../../../scripts/ply_subset.py)
   with recorded arguments, so it can be regenerated byte-for-byte while the
   source export is retained.
4. **Size.** At most a few MB per asset; anything larger belongs in the
   opt-in performance tier, not in git history.

## Layout

```
corpus/
  <asset-name>/
    <asset-name>-<derivation>.ply                 committed subset
    <asset-name>-<derivation>.ply.provenance.json machine-readable derivation record
    PROVENANCE.md                                 capture, training, and license record
```

`test_gaussian_ply_plugin.py` discovers every `corpus/*/*.ply`, reads the
expected Gaussian count and SH degree from the provenance record, and checks
the decoded stage semantically (finite values, positive scales, normalized
orientations, sigmoid-range opacities, extent containment). Corpus assets are
intentionally not declared in `openstrata.plugin.yaml`, so they are not
packaged into the distributed bundle.
